The stateActive-Binding

The stateActive-binding (short form sactive) tests if the router’s current state matches its own configuration and add a CSS-Class to its target element if the test succeeds or otherwise remove the class. By default the binding will use the class name active.

The primary use-case is to simplify the special appearance of navigation menus relying on routing-state, by having the active state’s menu button appear different, distinguishing it from the inactive items.

Aliases: sactive, stateActive and stateactive

Example

<a data-bind="sref: 'contacts', sactive: 'contacts'">Contacts</a>

wx.router.state({
    name: "home",
    views: {
        'main': "home"
    }
}).state({
    name: "contacts",
    views: {
        'main': "contact-list"
    }
})

wx.applyBindings();

wx.router.go("home");

The anchor will not bear the active class because the state does not match.

wx.router.go("contacts");

Now, the anchor will bear the active class because the state matches.

Parameters

There are two ways to use the sactive-binding:

Shorthand syntax

If you pass just a string, it is interpreted as a state name. The state-name will be substring-matched against the Router’s current state.

Full syntax

Alternatively, pass an object with the following properties:

  • name — the name of a state
  • params — an object that will have each of its properties compared against the router’s current state-params
  • cssClass - the name of one or more whitespace separated css-classes to apply to the element when the current router-state matches the binding’s own configuration
